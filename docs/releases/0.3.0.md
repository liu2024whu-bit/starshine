# Starshine Geo 0.3.0

Starshine Geo 0.3.0 turns the public project from a small fixed workflow core into a more
self-describing research preview. The release adds a reviewed declarative operator registry, a
machine-readable catalog, explicit reprojection, deterministic inspection and benchmark reports,
installed-wheel verification, and synthetic teaching cases. All implementation and validation
evidence in this release is maintained in the public repository and uses public or self-created data.

## Highlights

- one immutable operator registry defines executors, named inputs, parameter validators, defaults,
  JSON-compatible schemas, summaries, and output-CRS behavior;
- `starshine operators` and `operator_catalog()` publish the bounded operator contract without
  exposing Python callables or enabling dynamic imports;
- `reproject_features()` and the `reproject` workflow operation preserve properties and feature
  order while requiring explicit, non-conflicting source and target CRS metadata;
- `starshine inspect` and `inspect_feature_collection()` produce schema-checked structural reports
  containing counts, fields, CRS, bounds, and a deterministic digest without copying feature values;
- deterministic synthetic benchmarks cover buffer, dissolve, point summary, and a multi-step
  workflow while keeping semantic correctness separate from environment-specific timing;
- clean Python 3.10, 3.11, and 3.12 jobs install the exact CI-built wheel without a repository
  checkout and exercise the installed API and CLI;
- synthetic teaching fixtures demonstrate projected-CRS requirements, invalid geometry, and
  malformed properties with exact commands and stable diagnostics.

## Compatibility and upgrade notes

- Python 3.10, 3.11, and 3.12 remain supported and are tested from both editable source installs and
  the built wheel;
- GeoJSON remains the canonical in-memory contract;
- the optional GeoPackage adapter remains isolated behind the `geopackage` extra;
- Workflow version 1 remains current and now includes the `reproject` step;
- existing buffer, dissolve, and point-summary workflows remain valid when they satisfy the published
  Workflow v1 schema;
- operator discovery should use the public catalog rather than importing internal registry objects.

## Security and extension boundary

The declarative registry is a controlled extension point, not a plugin loader. Workflow JSON cannot
name modules, supply callables, execute Python, or invoke unregistered functions. Adding a new
operator still requires a public issue, implementation, `OperatorSpec`, Workflow schema branch,
stable diagnostics, focused tests, installed-wheel coverage, and normal pull-request review.

## Reproducibility and release evidence

The release candidate is required to pass:

- the tracked-file public-boundary audit;
- Ruff and the complete test suite on Python 3.10–3.12;
- the isolated GeoPackage round-trip job;
- deterministic benchmark verification and schema checking;
- package metadata, citation, changelog, README, and release-note consistency checks;
- wheel and source-distribution builds, Twine metadata validation, and archive inspection;
- non-editable installation and smoke execution of the exact built wheel on Python 3.10–3.12.

## Public source and data boundary

This release contains no private repository module, private database, proprietary dataset, textbook
or OCR content, internal deployment configuration, personal path, or unpublished research delivery
artifact. Examples, benchmark geometries, teaching failures, workflow fixtures, and GeoPackage
round-trip data are synthetic and maintained directly in this repository.

## Known limitations

- the public operator set remains intentionally small; clip and broader overlay/proximity operations
  are planned through future public issues;
- reprojection performs an explicitly requested coordinate transformation but does not choose an
  analysis CRS for the user;
- the operator registry does not yet accept third-party runtime plugins;
- no PostGIS adapter or web interface is included;
- third-party reproduction evidence remains pending until an actual external user supplies it.
