# Deterministic workflow graphs

Starshine can turn a validated data-free workflow plan into a stable graph for code review,
teaching, reports, CI, and future user interfaces. Graph construction does not read GeoJSON, open a
GeoPackage, or execute a spatial operator.

The graph layer deliberately reuses `plan_workflow()` rather than maintaining a second dependency
engine. Workflow validation, layer provenance, operation order, required external layers, produced
layers, and terminal outputs therefore remain single-sourced.

## Python API

```python
from starshine_geo import build_workflow_graph, render_workflow_mermaid

graph = build_workflow_graph(
    workflow,
    {"source", "mask"},
)
mermaid = render_workflow_mermaid(graph)
```

`build_workflow_graph()` returns a JSON-ready report conforming to:

```text
schemas/workflow-graph-v1.schema.json
```

The graph records:

- Workflow, Operator Catalog, and plan digests;
- declared, required, and unused external layers;
- produced and terminal layers;
- stable external-layer, operation, and produced-layer node identifiers;
- input and output edges with input-role labels;
- deterministic node and edge counts;
- a digest covering the complete graph body.

It does **not** include parameter values, feature content, file paths, or credentials.

## Command line

Render Mermaid text to standard output:

```bash
starshine graph examples/plan.workflow.json \
  --layer-name source \
  --layer-name mask
```

Write a machine-readable JSON graph:

```bash
starshine graph examples/plan.workflow.json \
  --layer-name source \
  --layer-name mask \
  --format json \
  --output examples/output/plan.graph.json
```

Write Mermaid text:

```bash
starshine graph examples/plan.workflow.json \
  --layer-name source \
  --layer-name mask \
  --output examples/output/plan.mmd
```

The output path cannot overwrite the Workflow file. Invalid workflows use the same stable text or
JSON diagnostics as `starshine validate` and `starshine plan`.

## Mermaid safety

Mermaid node identifiers are generated by Starshine and never copied from user layer names. Human
labels are escaped for ampersands, quotes, angle brackets, vertical bars, and newlines. The renderer
includes only layer names, operation names, step numbers, input roles, and terminal/unused status.

The tracked output `examples/plan.workflow.mmd` is generated from `examples/plan.workflow.json` and
shows the complete three-step example, including the unused external layer and terminal output.

## Scope

A graph explains intended structure. It does not prove data-dependent facts such as CRS equality,
geometry type compatibility, actual match counts, output field collisions, or whether an operation
will produce an empty result. Those checks remain in input validation and operator execution.

A graph is also not an execution manifest. Use:

- `starshine plan` for resolved defaults and detailed pre-execution metadata;
- `starshine graph` for a reviewable dependency view;
- `starshine run --manifest` for post-execution reproducibility evidence.
